# Рефакторинг source.py

## Цель
Упростить структуру кода, выделить повторяющиеся части в отдельные модули, улучшить читаемость и поддерживаемость.

## Структура после рефакторинга

```
source_amo_custom/
├── __init__.py              # Экспорт SourceAmoCustom
├── source.py                # Основной модуль (268 строк, было 734)
├── base_stream.py           # Базовые классы потоков (120 строк)
├── incremental_stream.py    # Базовый класс для инкрементальных потоков (72 строки)
├── token_manager.py         # Управление OAuth токенами (75 строк)
├── spec.py                  # Спецификация коннектора (60 строк)
├── schemas.py               # JSON схемы для потоков (180 строк)
└── constants.py             # Константы (12 строк)
```

## Выделенные модули

### 1. `constants.py`
- `FULL_LOAD_THRESHOLD` — порог для режима полной загрузки
- `MAX_RECORDS_PER_PAGE` — лимит записей на страницу
- `RATE_LIMIT_DELAY_SECONDS` — задержка между запросами
- Константы для backoff времени

### 2. `token_manager.py`
- `DatabaseTokenManager` — управление OAuth токенами через PostgreSQL
- Автоматическое обновление токенов
- Кеширование в памяти

### 3. `spec.py`
- `SPEC_DICT` — спецификация коннектора для Airbyte UI
- Все параметры подключения

### 4. `schemas.py`
- Функции для генерации JSON схем:
  - `get_leads_schema()`
  - `get_contacts_schema()`
  - `get_events_schema()`
  - `get_pipelines_schema()`
  - `get_custom_fields_schema()`
  - `get_users_schema()`
- Общая функция `get_custom_field_schema()` для переиспользования

### 5. `base_stream.py`
- `AmoStream` — базовый класс для всех потоков
  - Аутентификация через токены
  - Обработка ошибок и retry логика
  - Пагинация
  - Rate limiting
- `AmoFullRefreshStream` — базовый класс для справочников

### 6. `incremental_stream.py`
- `AmoIncrementalStream` — базовый класс для инкрементальных потоков
  - Управление state/cursor
  - Режимы Full Load / Incremental
  - Построение параметров запросов

### 7. `source.py` (упрощённый)
- Классы потоков данных (Leads, Contacts, Events, Pipelines, etc.)
- `SourceAmoCustom` — главный класс коннектора

## Преимущества

1. **Читаемость**: Каждый модуль отвечает за свою область
2. **Поддерживаемость**: Изменения в одном месте применяются ко всем потокам
3. **Тестируемость**: Модули можно тестировать независимо
4. **Переиспользование**: Базовые классы можно расширять для новых потоков
5. **Размер файлов**: `source.py` уменьшился с 734 до 268 строк
