## 3. Timeline синхронизации (РИСК CLOCK SKEW)

```
Сценарий: Время на L2 спешит на 5 минут

L2 Time:     10:00    10:05    10:10    10:15
             │────────│────────│────────│
Prod Time:   09:55    10:00    10:05    10:10
             │────────│────────│────────│

Data flow:
  L2 generates at 10:05 (has _synced_at = 10:05)
              ├─────────────────────────────────────────────┐
              │                                             │
  v_from_ts = 10:00 - 15 min = 09:45                       │
              ├────────────────────────────────────────┐    │
              │                                        │    │
  Sync 1 (10:00 PROD time):    ✅ 10:05 >= 09:45 (CAUGHT)  │
                               └─────────────────────────────┘
              ┌──────────────────────────────────────────────┐
              │                                              │
  Sync 2 (10:05 PROD time):    ❌ 10:10 >= 09:50 (MISSED!)
                               (because L2 time already moved to 10:10)
                               └─────────────────────────────────┘

РЕШЕНИЕ: Увеличить интервал с 15 мин на 1 HOUR
```

## 4. Ghost Busting Protection (ТЕКУЩИЙ РИСК)

```
СЦЕНАРИЙ: FDW обрывается на середине выборки

┌─────────────────────────────────────────────────────────────┐
│ PRODUCTION (имеет 5000 лидов)                               │
│                                                             │
│  amo_support_schema.concepta_leads: 5000 строк              │
└─────────────────────────────────────────────────────────────┘
                         │
                         │ Ghost Busting запустился
                         ▼
┌─────────────────────────────────────────────────────────────┐
│ FDW ВЫБОРКА (ОБРЫВАЕТСЯ)                                    │
│                                                             │
│  _meta_leads: 5 строк (вместо 5000 ❌)                       │
└─────────────────────────────────────────────────────────────┘
                         │
                         │ ТЕКУЩАЯ ПРОВЕРКА (НЕПРАВИЛЬНАЯ)
                         ▼
         IF (SELECT COUNT(*) FROM _meta_leads) = 0 AND 
            v_prod_leads_count > 100 THEN
              RAISE EXCEPTION
         ELSE
              👉 DELETE 4995 лидов! ❌ БЕЗ ЗАЩИТЫ!
         END IF

────────────────────────────────────────────────────────────────

РЕШЕНИЕ: Проверка на % потерь

         loss_percent = 100.0 * (5000 - 5) / 5000 = 99.9%
         IF loss_percent > 20 THEN
              RAISE EXCEPTION ✅ ЗАЩИЩЕНО!
         END IF
```

## 6. Трансформация данных (ТЕКУЩАЯ ЦЕПЬ)

```
L1 (Airbyte Raw)
    │
    ├─ is_deleted = NULL
    ├─ _synced_at = NOW() (момент синхронизации из AmoCRM)
    │
    ▼ TRIGGER
L2 (Analytics)
    │
    ├─ is_deleted = TRUE/FALSE (через Tombstone Shield)
    ├─ _synced_at = NOW() (момент обработки в L2)
    ├─ Индексы по _synced_at (НУЖНО ДОБАВИТЬ)
    │
    ▼ FDW SYNC (с интервалом 15 мин - НУЖНО 1 HOUR)
Production (amo_support_schema)
    │
    ├─ is_deleted = TRUE/FALSE (через Ghost Busting)
    ├─ PRIMARY KEY lead_id (НУЖНО ПРОВЕРИТЬ)
    ├─ ON CONFLICT (lead_id) DO UPDATE (ТРЕБУЕТ PRIMARY KEY)
    ├─ Логирование в sync_log
    │
    ▼
FINAL DATA (готово для бизнеса)
```

## 10. Дерево решений: Что делать при ошибке?

```
          ┌─ Синхронизация падает?
          │
          ├─ "relation does not exist"
          │  └─ Таблицы concepta_* не созданы
          │     └─ РЕШЕНИЕ: CRITICAL_FIXES.sql раздел 1
          │
          ├─ "there is no unique or exclusion constraint"
          │  └─ PRIMARY KEY отсутствует
          │     └─ РЕШЕНИЕ: CRITICAL_FIXES.sql раздел 2
          │
          ├─ "FDW Server unavailable"
          │  └─ L2 (Analytics) недоступна
          │     └─ РЕШЕНИЕ: Проверить подключение L2
          │
          └─ "Invalid statement or undefined function"
             └─ Старая версия функции
                └─ РЕШЕНИЕ: Использовать sync_*_smart_v2()

          ┌─ Данные не попадают на Production?
          │
          ├─ Проверить: SELECT * FROM amo_support_schema.sync_log
          │  └─ status = 'error'? → Читай выше
          │  └─ status = 'success' но данных нет? → Clock Skew
          │     └─ РЕШЕНИЕ: Синхронизировать время NTP
          │
          └─ Лиды исчезают из Production?
             └─ Ghost Busting удалил их
                └─ РЕШЕНИЕ: Восстановить из резервной копии

          ┌─ Система работает, но нет видимости?
          │
          └─ Мониторинг не настроен
             └─ РЕШЕНИЕ: MONITORING_AND_ALERTS.sql
```
