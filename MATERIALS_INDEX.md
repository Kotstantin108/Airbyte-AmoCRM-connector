## 📊 ЧТО НАЙДЕНО

### 🔴 Критичные проблемы (4)
1. **Несоответствие доменов** — таблицы concepta_* не созданы на Production
2. **Недостаточная проверка Ghost Busting** — может удалить 99% данных
3. **Отсутствие Primary Key** — ON CONFLICT упадёт с ошибкой
4. **Clock Skew между L2 и Prod** — пропуск данных в перекрытии

### 🟠 Высокие приоритеты (3)
5. Отсутствие индексов на `_synced_at` (L2)
6. Нет мониторинга ошибок
7. Нет проверки FDW перед Ghost Busting

### 🟡 Средние приоритеты (3)
8. Edge case обновлений (уже реализовано ✓)
9. Риск overflow для больших батчей
10. Отсутствие таймаута на долгие транзакции

### 🟢 Низкие приоритеты (2)
11. Версионирование схемы (nice-to-have)
12. Логирование размера FDW (nice-to-have)

**Как использовать:**
```bash

# На Analytics (L2) сервере
psql -h <analytics_host> -U postgres -d <analytics_db> -c "
CREATE INDEX idx_concepta_leads_synced_at ON prod_sync.concepta_leads (_synced_at);
CREATE INDEX idx_concepta_contacts_synced_at ON prod_sync.concepta_contacts (_synced_at);
..."
```
